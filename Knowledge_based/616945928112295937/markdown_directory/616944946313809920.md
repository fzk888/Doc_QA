**背景介绍**

什么是源码可视化开发？
----------------------

### 传统的前端应用构建过程

我们在搭建自己的 Vue 和 小程序 应用时，往往会做以下的事情：

1.  查看设计图，到 UI 库中找类似的组件

2.  复制组件代码到源码中，并修改源码

3.  不断重复上述的过程

看似是正常的流程，但这里面存在两个问题：

编写页面和组件对非专业前端和刚入门前端的同学来说是有一定的学习成本

在不同项目中可能存在相似的业务组件，每次需要复制原来的组件代码到新项目，对于有大量重复的业务场景或需要快速完成业务项目开发的同学来说效率有点低

### 基于标件进行可视化开发

针对上面的两个问题，总结了以下的解决方案：

-   建立标件体系，提供生产脚手架，规范标件生产流程和方式，支持开发自定义标件。

-   标件分为平台标件、我的标件。平台标件是同类型项目都可以进行拖拽使用的，我的标件，只在我所属的项目中才能进行使用。我的标件要提升到平台标件需要提交审核，审核通过即可变成平台标件。

-   提供可视化搭建能力，轻松通过标件拼装生产页面，并支持快速将标件插入至源码中

在gidc源码可视化开发中，标件分为三种类型：

-   基础标件：功能比较确定同时复杂度较高，例如时间选择器等，一般由 antdv
    等基础组件库提供，脚手架都已集成。

-   页面标件：完整的页面模板

-   业务标件：用于页面开发使用的业务组件，与业务耦合度较高。

每个标件在可视化拖拽生成页面时，可分为如下三种使用方式：

-   安装依赖的方式（npm install）

-   创建页面时，下载标件源码到项目公共路径（src/components/gidc/xxx）

-   创建页面时，下载标件的源码到创建页面文件所在同级目录

下面介绍下如何开发项目 和 开发标件。

**创建项目及项目标件库**

创建应用会创建两份应用程序代码

-   应用开发程序，主要开发项目业务主程序，平常开发项目的代码脚手架。

-   标件库程序，主要用于开发业务主程序需要用到的组件，在此开发完上传后，主应用开发程序中可以进行可视化拖拽，进行快速组装业务页面，并下载拖拽好的页面代码进行业务开发。
